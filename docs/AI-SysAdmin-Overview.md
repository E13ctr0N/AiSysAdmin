# AI-ассистент для администрирования Linux-серверов

## Что это

MCP-сервер на Python, который предоставляет AI-модели набор готовых инструментов для управления серверами через SSH. Работает через Claude Code (CLI/Desktop).

## Почему Python, а не просто инструкции в файле

Вся логика вынесена в Python-код: подключение по SSH, выполнение команд, **парсинг вывода в структурированный JSON**. AI-модель не пишет bash-команды и не разбирает текстовый вывод — она вызывает готовые функции и получает чистые данные.

Это даёт принципиальное преимущество:

| | Без MCP (MD-инструкции) | С MCP (Python) |
|---|---|---|
| Модель сама пишет | `ssh root@host "free -b \| awk..."` | `system_info(server="main")` |
| Модель получает | сырой текст терминала | `{"cpu_cores": 1, "memory": {"total": 1003831296, ...}}` |
| Модель должна | знать Linux, парсить вывод, помнить синтаксис | понять JSON и ответить пользователю |
| Требуемый уровень модели | Opus / GPT-4 | **Haiku / Sonnet — достаточно** |

**Сложная работа (SSH, парсинг, валидация, бэкапы) — в Python. Простая работа (прочитать JSON, ответить человеку) — в модели.**

## Экономический эффект

| Модель | Стоимость (API) | Скорость | Справится? |
|---|---|---|---|
| Opus | $$$ | медленнее | да |
| Sonnet | $$ | быстро | **да** |
| Haiku | $ | очень быстро | **да** |

Разница в стоимости Opus vs Haiku — **~60x**. При рутинных задачах (проверь диск, перезапусти сервис, покажи логи) Haiku справится так же, потому что вся сложность — в Python-коде, а не в рассуждениях модели.

## Что умеет (22 инструмента)

**Мониторинг**: системная информация, диски, процессы, порты, сервисы

**Управление**: выполнение команд, установка пакетов, управление сервисами, редактирование конфигов с автобэкапом

**Безопасность**: комплексный аудит, фаервол, проверка обновлений

**Docker**: контейнеры, образы, логи, Compose-операции

**Обслуживание**: бэкапы, cron-задачи, генерация отчётов

## Демонстрация

Одна сессия (~15 минут), боевой сервер:

- Полный аудит безопасности — выявлено 6 проблем
- Установлен и настроен fail2ban (16 800+ попыток брутфорса)
- Включён UFW — 9 правил под конкретные сервисы
- Ужесточены SSH и kernel-параметры
- Очищено 900 МБ логов, настроены лимиты и ротация

AI сам проверил наличие SSH-ключей перед отключением паролей — и остановился, чтобы не потерять доступ.

## Требования

- Claude Code (бесплатный CLI/Desktop)
- API-ключ Anthropic или подписка Pro/Max
- Python 3.11+ на рабочей станции
- SSH-доступ к серверам

## Архитектура

```
Администратор -> Claude Code -> MCP-протокол -> Python-сервер -> SSH -> Linux-сервер
     текст         AI-модель      JSON-RPC       Paramiko        команды
```

Модель — заменяемый компонент. Python-сервер — ядро системы.
